Full-Stack Interview Kit Tester

This page describes the two versions of the application and how to test them.

1. Clean Version (Skeleton)

Status: Fully Functional

This version has all bugs fixed and follows best practices.

  • Deterministic pagination.
  • Correct integration contract (data key).
  • Optimized suggestion engine.
  • Immutable frontend sorting.
cd unzipped_skeleton && make test

2. Interview Version

Status: Contains 8+ Intentional Bugs

This version is for the candidate to debug.

  • BE: Unstable sort & off-by-one pagination.
  • FE: State mutation & stale effects.
  • Integration: Key mismatch (items vs data).
  • Algo: O(N*M) complexity trap.
cd unzipped_interview && make test

How to run the automated test script:

python3 test_both_apis.py
